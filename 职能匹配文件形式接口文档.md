# 职能匹配文件形式智能体接口文档

## 1. 接口概述

本文档说明“职能匹配文件形式”智能体的 HTTP 调用方式。

本接口采用 **多文件上传模式**，文件直接作为 `multipart/form-data` 参数提交给 `scene_gateway`。

## 2. 接口地址

```text
POST http://2.142.92.117:30486/scene_gateway/ad441c290aef406fa9350344f513461c
```

接口描述：

```text
职能匹配文件形式
```

请求类型：

```text
HTTP / multipart/form-data
```

## 3. 请求头

| 参数英文名 | 参数中文名 | 参数位置 | 是否必填 | 说明 |
|---|---|---|---|---|
| AuthToken | 授权密钥 | header | 是 | 平台授权密钥 |

AuthToken：f2fa523939374d8392c9eb50a6dcb245

## 4. 请求参数

请求体类型：`multipart/form-data`

| 参数英文名 | 参数中文名 | 参数类型 | 是否必填 | 说明 |
|---|---|---|---|---|
| keyword | 关键词 | string | 是 | 智能体输入关键词，例如：`分析` |
| requestId | 会话ID | string | 是 | 单次调用唯一会话ID，建议每次调用生成新的值 |
| sceneKey | 应用ID | string | 否 | 当前智能体应用ID，
| dialogId | 每一轮对话ID | string | 否 | 多轮对话时使用；单轮调用可不传 |
| zhineng | zhineng | file | 否 | 职能相关文件，直接上传文件（文件类型 .xlsx） |
| mulu | mulu | file | 否 | 目录/匹配对象相关文件，直接上传文件（文件类型 .xlsx） |



## 6. 调用流程

```text
本地选择文件
  -> 构造 multipart/form-data
  -> keyword、requestId 作为普通表单字段
  -> zhineng、mulu 作为文件字段
  -> 直接 POST 到 scene_gateway
  -> 获取智能体返回结果
```

## 7. multipart/form-data 字段示例

```text
keyword = 分析
requestId = 20260623001
sceneKey = 8a9640a680b94cee85780703e1cd0de5
dialogId =
zhineng = @职能文件.xlsx
mulu = @目录文件.xlsx
```

## 8. CURL示例
```
curl.exe -X POST "http://2.142.92.117:30486/scene_gateway/ad441c290aef406fa9350344f513461c" -H "AuthToken: f2fa523939374d8392c9eb50a6dcb245" -F "keyword=分析" -F "requestId=20260623001" -F "sceneKey=ad441c290aef406fa9350344f513461c" -F "dialogId=" -F "zhineng=@C:\Users\15533\Desktop\数据局内设机构和职能.xlsx;type=application/vnd.openxmlformats-officedocument.spreadsheetml.sheet" -F "mulu=@C:\Users\15533\Desktop\市数据局目录(1) - 副本.xlsx;type=application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"
```
